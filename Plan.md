
# ОПИСАНИЕ ПЛАНА ПО ПРОВЕРКЕ И АВТОМАТИЗАЦИИ ПРИЛОЖЕНИЯ «В ХОСПИСЕ»

## 1. ПОДГОТОВИТЕЛЬНЫЙ ЭТАП

### Анализ требований:

#### Функциональные требования:
- Управление претензиями: 
  - Добавление, просмотр, редактирование и удаление претензий.
- Новостная сводка: 
  - Предоставление актуальных новостей и информации о хосписе.
- Цитаты: 
  - Просмотр тематических цитат.

### Выбор инструментов:
- Android Studio
- Espresso
- Git
- Allure

## 2. ПЛАНИРОВАНИЕ ТЕСТИРОВАНИЯ

### Определение тестовых сценариев:

#### Управление претензиями:
- Добавление, просмотр, редактирование и удаление претензий.

#### Новостная сводка:
- Просмотр, добавление, редактирование, удаление и обновление новостей.

#### Цитаты:
- Отображение цитат и просмотр коллекции.

### План автоматизации:
- Авторизация.
- Тесты на управление претензиями.
- Тесты на новостную сводку.
- Тесты на отображение цитат.

## 3. ТЕСТИРОВАНИЕ ВРУЧНУЮ

### Функциональное тестирование:
- Проверка каждой функции вручную в соответствии с заданными сценариями.

## 4. РАЗРАБОТКА АВТОТЕСТОВ

### Выбор тестовых сценариев для автоматизации:

#### 4.1. Тестовые сценарии для автоматизации управления претензиями:

А) Добавление новой претензии.
Б) Редактирование существующей претензии.
В) Удаление претензии.

#### 4.2. Тестовые сценарии для автоматизации новостной сводки:

А) Просмотр списка новостей.
Б) Добавление новой новости.
В) Редактирование существующей новости.
Г) Удаление новости.
Д) Обновление новостей.

#### 4.3. Тестовые сценарии для автоматизации цитат:

- Отображение коллекции цитат.

## 5. ЗАПУСК ТЕСТОВ

### Запуск автотестов:
- Проведение автоматизированных тестов на приложении для повторного тестирования.

### Анализ результатов:
- Оценка результатов тестирования, выявление проблем и ошибок.

## 6. ОТЧЕТНОСТЬ И ДОРАБОТКА

### Составление отчетов:
- Создание отчета о результатах тестирования.

### Работа над ошибками:
- Исправление выявленных проблем и ошибок, повторное тестирование исправлений.
